zcli

a zcash wallet that works like you do

derive a zcash wallet from your ssh key. no seed phrases, no interactive setup. one binary, one key, done.

install how it works

what it does

ssh key = wallet

your ed25519 key deterministically derives a zcash spending key. same key, same wallet, same addresses, every time.

agent-friendly

no prompts, no GUI. headless processes sync, check balance, and send transactions with env vars and JSON output.

privacy by default

trial decryption happens locally. the server never learns your addresses. transactions are shielded with halo2 proofs.

verified chain

ligerito proofs verify 1.5M+ headers. nomt merkle proofs bind every note and nullifier. cross-verified against independent nodes. run zcli verify to see it. details

zcli -i ~/.ssh/id_ed25519 address          # show addresses
zcli sync                                   # scan chain
zcli balance                                # check funds
zcli send 0.01 u1...                        # send shielded
zcli shield                                 # t-addr → shielded
zcli verify                                 # verify chain proofs
zcli notes --json                           # list UTXOs

all commands accept --json for machine output. set ZCLI_IDENTITY and ZCLI_ENDPOINT as env vars.

cold storage with zigner

zigner is an air-gapped hardware signer for zcash. it runs on an ESP32 with a camera and display - no network, no USB data. keys never leave the device.

zigner generates keys, signs transactions offline via QR codes
→
zcli imports viewing key, syncs chain, builds unsigned tx, broadcasts signed tx 
zcli import-fvk --cam         # scan zigner's FVK QR
zcli sync --wallet zigner     # sync the watch wallet
zcli send 1.0 u1... --cam     # build tx, scan signed QR back

zigner.rotko.net - open source, open hardware.
